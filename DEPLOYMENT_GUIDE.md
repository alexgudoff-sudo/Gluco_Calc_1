## Быстрый старт

### Требования

Перед началом убедитесь, что установлены:

- **Node.js 18+** ([скачать](https://nodejs.org/))
- **pnpm** или **npm** (менеджер пакетов)
- **Git** (опционально)

### Установка за 3 шага

```bash
# 1. Распакуйте архив и перейдите в папку
unzip gluco-calc.zip
cd gluco-calc

# 2. Установите зависимости
pnpm install

# 3. Запустите приложение
pnpm dev
```

Приложение откроется в браузере на `http://localhost:8081`.

## Система хранения данных

### Что сохраняется

Приложение использует **AsyncStorage** для локального сохранения всех пользовательских настроек:

| Данные | Описание | Значение по умолчанию |
|--------|---------|----------------------|
| Коэффициент чувствительности углеводов | Как много ммоль/л глюкозы поднимается на 10g углеводов | 2.2 ммоль/л |
| Коэффициент чувствительности инсулина | Как много ммоль/л глюкозы снижается на 1 единицу инсулина | 1.8 ммоль/л |
| Выбранная тема | Светлая или тёмная тема интерфейса | Светлая |
| Выбранный язык | Русский или английский | Русский |

### Где хранятся данные

**На веб-браузере:** Данные хранятся в `localStorage` браузера. Они сохраняются автоматически и остаются даже после закрытия браузера.

**На мобильном устройстве (Expo Go):** Данные хранятся в приложении Expo Go и сохраняются между сеансами.

**На iPhone (PWA):** Данные хранятся в локальном хранилище PWA приложения и сохраняются между запусками.

### Резервное копирование данных

Так как данные хранятся локально, рекомендуется периодически сохранять ваши коэффициенты:

1. Откройте приложение
2. Нажмите кнопку **⚙️ (Настройки)** в центре экрана
3. Запишите текущие значения коэффициентов
4. Сохраните их в безопасном месте (заметки, документ и т.д.)

## Локальный запуск

### На компьютере (веб-браузер)

Это самый быстрый способ для тестирования:

```bash
pnpm dev
```

Приложение откроется на `http://localhost:8081`. Все изменения в коде применяются автоматически (hot reload).

### На физическом устройстве (iOS/Android)

Для тестирования на реальном устройстве:

1. **Установите Expo Go:**
   - iOS: [App Store](https://apps.apple.com/app/expo-go/id982107779)
   - Android: [Google Play](https://play.google.com/store/apps/details?id=host.exp.exponent)

2. **Запустите dev сервер:**
   ```bash
   pnpm dev
   ```

3. **Отсканируйте QR-код:**
   - iOS: Откройте камеру и отсканируйте QR-код из терминала
   - Android: Откройте Expo Go и отсканируйте QR-код

4. **Приложение загрузится на вашем устройстве**

### На iOS симуляторе (требует macOS)

```bash
pnpm ios
```

### На Android эмуляторе

```bash
pnpm android
```

## PWA на iPhone

**Progressive Web App (PWA)** позволяет добавить приложение на главный экран iPhone без Expo Go. Приложение будет работать как обычное приложение, но в браузере.

### Требования

- iPhone с iOS 11.3 или выше
- Safari браузер
- Интернет соединение

### Шаг 1: Запуск веб-версии

```bash
pnpm dev
```

Откройте браузер на адресе `http://localhost:8081`.

### Шаг 2: Добавление на главный экран

1. Откройте приложение в **Safari** на iPhone
2. Нажмите кнопку **Поделиться** (значок со стрелкой внизу)
3. Прокрутите вниз и выберите **"На экран Домой"**
4. Введите имя приложения (например, "GlucoCalc")
5. Нажмите **"Добавить"**

Приложение появится на вашем главном экране как обычное приложение!

### Особенности PWA на iPhone

| Функция | Поддержка |
|---------|-----------|
| Добавление на главный экран | ✅ Да |
| Полноэкранный режим (без адресной строки) | ✅ Да |
| Локальное хранилище данных | ✅ Да |
| Автоматические обновления | ⚠️ При перезагрузке браузера |
| Работа без интернета | ⚠️ Ограниченная (требует Service Worker) |
| Отправка на App Store | ❌ Нет (используйте EAS Build вместо этого) |

### Обновление PWA

Когда вы обновляете код приложения, PWA на iPhone обновляется автоматически при следующем открытии. Если изменения не видны:

1. Закройте приложение полностью
2. Откройте его снова

## Развёртывание на сервер

Для использования PWA на реальном сервере (чтобы другие люди могли использовать приложение):

### Шаг 1: Сборка веб-версии

```bash
pnpm build
```

Это создаст папку `dist/` с готовым приложением.

### Шаг 2: Загрузка на сервер

Выберите один из рекомендуемых хостингов:

#### Vercel (рекомендуется для Expo)

1. Создайте аккаунт на [vercel.com](https://vercel.com)
2. Подключите ваш GitHub репозиторий
3. Vercel автоматически соберёт и развернёт приложение
4. Ваше приложение будет доступно на `https://your-app.vercel.app`

#### Netlify

1. Создайте аккаунт на [netlify.com](https://netlify.com)
2. Перетащите папку `dist/` в область "Deploy"
3. Netlify загрузит приложение
4. Ваше приложение будет доступно на уникальном URL

#### GitHub Pages (бесплатно)

1. Создайте репозиторий на GitHub
2. Включите GitHub Pages в настройках репозитория
3. Загрузите содержимое папки `dist/` в ветку `gh-pages`
4. Ваше приложение будет доступно на `https://your-username.github.io/gluco-calc`

### Шаг 3: Проверка HTTPS

PWA требует HTTPS для работы. Все рекомендуемые хостинги предоставляют HTTPS автоматически.

### Шаг 4: Поделитесь ссылкой

После развёртывания пользователи смогут:

1. Открыть ваше приложение в браузере
2. Добавить его на главный экран iPhone (или Android)
3. Использовать как обычное приложение

## Решение проблем

### Приложение не загружается в браузере

**Проблема:** Белый экран или ошибка подключения

**Решение:**

1. Проверьте, что dev сервер запущен: `pnpm dev`
2. Убедитесь, что порт 8081 не занят:
   ```bash
   # macOS/Linux
   lsof -i :8081
   
   # Windows
   netstat -ano | findstr :8081
   ```
3. Если порт занят, используйте другой:
   ```bash
   EXPO_PORT=3000 pnpm dev
   ```
4. Очистите кэш браузера (Ctrl+Shift+Delete)

### Ошибка при установке зависимостей

**Проблема:** `npm ERR!` или `pnpm ERR!`

**Решение:**

```bash
# Удалите папку node_modules и lock файл
rm -rf node_modules pnpm-lock.yaml

# Переустановите зависимости
pnpm install
```

### QR-код не сканируется на iPhone

**Проблема:** Камера не распознаёт QR-код

**Решение:**

1. Убедитесь, что ваш телефон подключён к той же сети Wi-Fi, что и компьютер
2. Попробуйте использовать туннель:
   ```bash
   pnpm dev --tunnel
   ```
3. Если туннель не работает, используйте локальный IP:
   ```bash
   # Найдите IP адрес вашего компьютера
   ipconfig getifaddr en0  # macOS
   hostname -I            # Linux
   
   # Откройте в браузере на телефоне
   http://YOUR_IP:8081
   ```

### Данные не сохраняются

**Проблема:** Коэффициенты сбрасываются при перезагрузке

**Решение:**

1. Проверьте, что браузер позволяет использовать localStorage:
   - Откройте DevTools (F12)
   - Перейдите в Application → Local Storage
   - Убедитесь, что там есть данные приложения

2. Если localStorage отключён, включите его в настройках браузера

3. На iPhone (PWA) очистите кэш приложения:
   - Удалите приложение с главного экрана
   - Добавьте его снова

### Приложение медленно работает

**Проблема:** Лаги, зависания, медленная прокрутка

**Решение:**

1. Закройте другие приложения и вкладки браузера
2. Перезагрузите dev сервер:
   ```bash
   # Нажмите Ctrl+C в терминале
   # Затем запустите снова
   pnpm dev
   ```
3. Очистите кэш браузера
4. Попробуйте использовать другой браузер (Chrome, Firefox, Safari)

### Приложение не обновляется при изменении кода

**Проблема:** Hot reload не работает

**Решение:**

1. Убедитесь, что файл сохранён (Ctrl+S)
2. Проверьте, что нет ошибок в консоли браузера (F12)
3. Перезагрузите страницу вручную (Ctrl+R)
4. Перезагрузите dev сервер:
   ```bash
   # Нажмите Ctrl+C
   pnpm dev
   ```
